# Paytm Web App Clone (HTML & CSS Only)

## 1. Project Introduction

This project is a static **Paytm Web Application UI clone** developed using **only HTML and CSS**.  
The primary objective of this project is to replicate the **visual design, layout structure, and navigation flow** of the Paytm application in the form of a **desktop-oriented web interface**, without using JavaScript, frameworks, or backend technologies.

This project is intended strictly for **educational, academic, and portfolio demonstration purposes** and does not implement any real payment processing or dynamic business logic.

---------------------------------------------------------------------------------------------------------

## 2. Project Objectives

- To understand and implement real-world UI layout patterns
- To practice multi-page navigation using pure HTML
- To design reusable and scalable CSS architecture
- To build a clean, professional card-based UI
- To simulate Paytm’s core interface sections visually
- To demonstrate front-end structuring skills without JavaScript

----------------------------------------------------------------------------------------------------------

## 3. Technologies Used

- **HTML5**
  - Page structure and semantic layout
  - Navigation between pages
  - Content organization

- **CSS3**
  - Flexbox and Grid layouts
  - Card-based UI components
  - Responsive spacing and alignment
  - Color theming and typography
  - Shadows and rounded corners

No external libraries or frameworks were used in this project.

-----------------------------------------------------------------------------------------------------------

## 4. Project Features

- Multi-page web application structure
- Home page with wallet balance and quick actions
- Recharge and bill payment interface
- Profile page with user information
- Reusable navigation bar across all pages
- Image-based icons and logo (placeholders)
- Clean Paytm-inspired color palette
- Desktop web application layout
- Consistent design across all pages

## 6. How the Application Works

- Each page is a separate HTML file
- Navigation is handled using standard anchor tags
- All pages share a common stylesheet for consistency
- Icons and logos are loaded as images from the assets folder
- No client-side scripting or dynamic behavior is implemented

-----------------------------------------------------------------------------------------------------

## 7. Limitations

- No real payment functionality
- No backend or database integration
- No form validation or dynamic state handling
- Static content only
- Designed primarily for desktop view
